Imagem dinâmica

No Gimp fizemos uma simulação de três tamanhos de imagens.

  • Imagem para Celular: Tamanho 300x300 px.
  • Imagem para tablet: Tamanho 700x700 px.
  • Imagem para computadores tvs etc...: Tamanho 1000x1000 px.

Todas com 80.000 px de resolução.

Tente abrir esse site em vários dispositivos diferentes ou simplesmente aumente e diminua o tamanho do seu navegador.

Vamos criar uma tag nova <picture></picture> Essa tag parece com a tag imagem, ela tem o img dentro mas ela permite que você crie vários source, várias fontes, isto é, vai aparecer uma imagem aqui... mas qual imagem? Dizemos a imagem que vai aparecer com o código que colocamos dentro da tag <picture></picture>

            
            <picture>
                <img src="imagens/foto-g.png" alt="Imagem flexível">
            </picture>

Dentro da tag <picture></picture> vamos criar uma tag img src= "" alt="" em src= colocar a imagem maior ("imagem/foto-g.png") em alt escrever "imagem flexível"

Acima da tag img vamos criar outra tag chamada <source></source> no formato média:type que já sugere a largura mínima mas podemos começar com a largura máxima é só mudar para "(max-width: )" você coloca o tamanho da imagem grande "(max-width: 1000px)" O tamanho máximo sendo mil (Coloque sempre 50px a mais pra evitar a barra de rolagem), podemos carregar a foto M colocando o nome da imagem m em srcset="imagens/foto-m.png" como ela é menor que 1000px ele vai carregar a foto m. Em type= "image/ coloca o tipo da imagem no caso png.

Como incluir a imagem p? Acima da imagem M (tem que ser acima!) vamos criar outro source do mesmo tipo media:type=. Em média="(max-width: 750px)" em scrset="imagem/foto-p.png" e em type="image/png"

        
            <picture>
                <img src="imagens/foto-g.png" alt="Imagem flexível">
            </picture>
        
    
        
            <picture>
            <source media="(max-width: 1050px)" srcset="imagens/foto-m.png" 
        type="image/png"> <img src="imagens/foto-g.png" alt="Imagem flexível">
        
    
        
            <picture>
                <source media="(max-width: 750px)" srcset="imagens/foto-p.png" 
    type="image/png">
                <source media="(max-width: 1050px)" srcset="imagens/foto-m.png" 
 type="image/png">
                <img src="imagens/foto-g.png" alt="Imagem flexível">
            </picture>

Importante! Sempre vai colocando os source de baixo para cima da imagem maior para a menor

Reproduzindo Áudio

Vamos aprender a reproduzir áudios em HTML5.

Colocando áudio no site

precisa tomar bastante atenção com o tamnho do arquivo, arquivo mp3 com máxima qualidade geram arquivos muito grandes, assim como as imagem, porém áudios grandes são maiores ainda. Tudo que vimos sobre imagem pode ser tranferido para mídias e vídeos etc.

Existem duas maneiras de adicionar áudio.

Reproduzindo áudio

Vamos aprender a reproduzir áudios em HTML5.

Escreva a tag audio que ja vai pedir o parâmetro src="" aperta as teclas Ctrl + espaço e seleciona o áudio que você quer reproduzir. Em seguida coloque mais dois parâmetro o controls e o autoplay.

            
            <audio>
                src="midia/Love Aside - Patrick Patrikios.mp3" controls autoplay>
            </audio>

Muitos navegadores não tem compatibilidade com mp3 por exemplo, mas podemos usar outros arquivos. Mas pra isso você vai ter que ter o formato MP3, WAV, OGG. Se quiser ter compatibilidade total com todos os navega
dores mesmo assim ainda ficar meio pendurado no autoplay que, as vezes, não funciona você tem que ter uma outra forma de adicionar o áudio.

Outra maneira de colocar áudio

            
                <audio preload="metadata" controls loop>
                    <source src="midia/Clean and Dance - An Jone.mp3" type="audio/mpeg">
                    <source src="midia/guanacast-33.mp3" type="audio/ogg">
                    <source src="midia/guanacast-33.wav" type="audio/wav">
                </audio>

Lembre-se que a ordem dos arquivos é muito importante, o que você colocar primeiro ele vai tentar rodar, se ele não conseguir ele vai tentar carregar o próximo da lista de cima pra baixo.

Se ele não conseguir reproduzir nenhum deles significa que seu navegador não consegue reproduzir áudio.

Clique aqui para baixar o arquivo mp3

Podemos incluir alguns parâmetros como o preload="" que vai vai ter três opções auto, metadata e none

O padrão é auto e é meio perigoso pq ele só vai considerar que o site terminou de carregar depois que você carregar todo áudio também, no caso de um arquivo grande como o do podcast com quase 32 megas é muito pesado, se tentar carregar no celular vai complicar.

Se você colocar nada ou colocar auto é umas das piores coisas que você pode fazer no seu site. metadata vai carregar só o nome do arquivo, algumas informações tipo data, duração etc. E none ele não vai carregar nada, só quando o usuário apertar play.

Existe também o loop que é o seguinte quando o áudio termina ele recomeça o áudio.

Obs.: tente evitar arquivos WAV pois são arquivos extremaente grandes e pesados. Falando tanto de imagem, vídeo ou áudio, qualidade top é pra arquivo feito download não para arquivo de navegação
